## Color/Format Summary

| Color Var | ANSI | Semantic Use |
|-----------|------|-------------|
| `$BRAND` | Bold Cyan `\033[1;36m` | Headers, branding, primary labels |
| `$SUCCESS` | Green `\033[0;32m` | Checkmarks, "done", positive outcomes |
| `$DIM` | Gray `\033[0;90m` | Secondary info, hints, paths |
| `$WARN` | Yellow `\033[0;33m` | Warnings, available updates, caution |
| `$ERROR` | Red `\033[0;31m` | Errors, failures, missing dependencies |
| `$BOLD` | Bold `\033[1m` | Emphasis, values, names |
| `$RESET` | Reset `\033[0m` | Return to normal |

## Formatting Patterns

| Pattern | Where Used |
|---------|-----------|
| `printf "  ${SUCCESS}✓${RESET} %s"` | Doctor checks, reload/update confirmations |
| `printf "  ${ERROR}✗${RESET} %s"` | Doctor fails, auth errors, missing deps |
| `printf "  ${WARN}⚠${RESET} %s"` | Doctor warnings, updates available |
| `printf "   ${DIM}[N/M]${RESET} %-40s"` | Step progress indicators |
| `printf "  ${BRAND}Doey — %s${RESET}"` | Section headers |
| `printf "  ${DIM}Label${RESET} ${BOLD}value${RESET}"` | Key-value info lines |
| `read -rp "  ... (y/N) " var` | y/N confirmation prompts |
| `read -rp "  > " var` | Free-form input (menu) |
| `read -rs var` | Secret input (API token) |
| `cat << 'HEREDOC'` | Help text, banners, ASCII art |
