# Charmbracelet Ecosystem Inventory for Doey CLI Upgrade

## Summary

Charmbracelet is a Go-focused org (~54 repos) building elegant CLI tools and TUI libraries. For a **bash-based CLI like Doey**, the primary tool is **gum** — a standalone binary that wraps the entire ecosystem's styling into shell-scriptable subcommands. Most other Charmbracelet tools are Go libraries (useful only if rewriting in Go) or specialized CLIs (git servers, AI agents, etc.) that aren't relevant to a CLI launcher upgrade.

---

## Tier 1: Directly Usable from Bash Scripts

### gum — Interactive prompts and styled output for shell scripts

- **What:** Standalone CLI binary that provides interactive prompts, spinners, confirmations, styled text, tables, pagers, file pickers, and more — all from bash
- **Install:** `brew install gum` / `go install github.com/charmbracelet/gum@latest` / binary download from GitHub releases
- **Type:** Standalone CLI binary (no Go code needed)
- **Relevance:** **PRIMARY TOOL** — covers 90% of what Doey needs for a luxury CLI upgrade

#### Complete Subcommand Reference

| Subcommand | Purpose | Example |
|------------|---------|---------|
| `confirm` | Yes/no confirmation prompt; exits 0 (yes) or 1 (no) | `gum confirm "Deploy to prod?" && deploy.sh` |
| `choose` | Pick one or more items from a list | `CHOICE=$(echo -e "alpha\nbeta\ngamma" \| gum choose)` |
| `filter` | Fuzzy-find/filter from piped input | `PICK=$(ls \| gum filter --placeholder "Search files...")` |
| `input` | Single-line text input with placeholder | `NAME=$(gum input --placeholder "Project name")` |
| `write` | Multi-line text input (Ctrl+D to finish) | `NOTES=$(gum write --placeholder "Describe the bug...")` |
| `spin` | Show spinner while a command runs | `gum spin --spinner dot --title "Building..." -- make build` |
| `table` | Display CSV as table, return selected row | `cat data.csv \| gum table --widths 20,10,30` |
| `format` | Render markdown, code, template, or emoji strings | `gum format '# Hello' '**bold text**' ':rocket: launched'` |
| `style` | Apply border, padding, margin, color to text | `gum style --border double --foreground 212 --padding "1 2" "Status: OK"` |
| `log` | Leveled, structured logging output | `gum log --level info --structured "Task complete" key value` |
| `file` | Interactive file picker from filesystem tree | `FILE=$(gum file /path/to/dir)` |
| `pager` | Scrollable pager for long content | `cat long-output.txt \| gum pager` |
| `join` | Join text blocks horizontally or vertically | `gum join --horizontal "$(gum style --border normal 'Left')" "$(gum style --border normal 'Right')"` |

#### Key gum Flags (cross-cutting)

- `--header "text"` — header text above prompts
- `--cursor.foreground "212"` — customize cursor color
- `--selected.foreground "212"` — customize selected item color
- `--timeout 10s` — auto-dismiss after timeout
- `--limit N` / `--no-limit` — multi-select for choose/filter
- Spinner types: `line`, `dot`, `minidot`, `jump`, `pulse`, `points`, `globe`, `moon`, `monkey`, `meter`, `hamburger`

#### Doey-Specific Use Cases

| Doey Feature | gum Subcommand | Notes |
|--------------|----------------|-------|
| Team selection on launch | `gum choose` | Pick team config from available .team.md files |
| Worker count prompt | `gum input --placeholder "Workers (2-8)"` | With validation in bash |
| Confirm destructive ops | `gum confirm "Kill all sessions?"` | Replace bare `read -p` prompts |
| Progress during setup | `gum spin --spinner dot --title "Spawning workers..."` | Wrap tmux session creation |
| Display task list | `gum table` | Render .doey/tasks/ as formatted table |
| Show docs/help | `gum format` or pipe to `gum pager` | Rich markdown rendering in terminal |
| Styled status banners | `gum style --border double --align center` | Replace raw echo with styled boxes |
| Log output in hooks | `gum log --level info "Hook fired"` | Structured, colored log lines |
| File picker for configs | `gum file ~/.config/doey/` | Browse and select config files |
| Layout composition | `gum join --horizontal` | Side-by-side info panels |

### glow — Render markdown in terminal

- **What:** Standalone CLI for rendering markdown with syntax highlighting and styles
- **Install:** `brew install glow` / `go install github.com/charmbracelet/glow@latest`
- **Type:** Standalone CLI binary
- **Relevance:** Medium — useful for rendering help docs, task descriptions, CLAUDE.md previews

#### Key Usage

```bash
glow README.md                              # Render file
glow -w 80 docs/context-reference.md        # Wrap at 80 cols
echo "# Title" | glow -                     # Render from stdin
glow -p README.md                           # Open in pager mode
glow -s dark README.md                      # Force dark style
```

#### Doey Use Cases

- `doey help` could render docs/ markdown beautifully
- Task descriptions in .doey/tasks/ rendered with glow
- Info panel could use glow for formatted output

### freeze — Generate images of terminal output

- **What:** CLI tool that generates SVG/PNG/WebP screenshots of code and terminal output
- **Install:** `brew install freeze` / `go install github.com/charmbracelet/freeze@latest`
- **Type:** Standalone CLI binary
- **Relevance:** Low-medium — useful for docs/demos, not runtime

#### Key Usage

```bash
freeze --execute "doey status"              # Capture command output as image
freeze main.go --lines 1,5 --output code.svg  # Capture specific lines
cat output.txt | freeze --language bash     # Pipe content
freeze --interactive                        # Interactive customization TUI
tmux capture-pane -p | freeze               # Screenshot tmux pane
```

#### Doey Use Cases

- Generate screenshots for documentation automatically
- `doey screenshot` command to capture session state
- CI/CD demo generation

### vhs — Terminal GIF/video recorder

- **What:** Write terminal recordings as code using `.tape` files, outputs GIF/MP4/WebM
- **Install:** `brew install vhs` (also needs `ttyd` and `ffmpeg`)
- **Type:** Standalone CLI binary
- **Relevance:** Low — documentation/demo tooling only, not runtime

#### Key Usage

```bash
# demo.tape file:
# Output demo.gif
# Set FontSize 14
# Set Width 1200
# Set Height 600
# Type "doey"
# Enter
# Sleep 3s
# Type "Hello team"
# Enter
# Sleep 5s

vhs demo.tape                               # Record from tape file
vhs record                                  # Interactive recording
vhs validate demo.tape                      # Validate tape syntax
```

#### Tape Commands

`Output`, `Set`, `Type`, `Enter`, `Sleep`, `Ctrl+C`, `Up`, `Down`, `Tab`, `Show`, `Hide`, `Screenshot`, `Copy`, `Paste`, `Require`, `Source`

#### Doey Use Cases

- Automated demo GIF generation for README
- CI-reproducible recording of Doey sessions
- Integration test visualization

## Tier 2: Go Libraries (Require Go Code)

These are **not usable from bash** but are what powers gum and other Charm CLIs internally. Only relevant if Doey ever gets a Go-based TUI component.

### lipgloss — Terminal styling library

- **What:** CSS-like style definitions for terminal layouts (colors, borders, padding, alignment)
- **Install:** `go get github.com/charmbracelet/lipgloss`
- **Type:** Go library only
- **Note:** gum's `style` and `join` subcommands expose lipgloss functionality to bash. No need to use lipgloss directly unless writing Go.

### glamour — Markdown rendering library

- **What:** Stylesheet-based markdown to ANSI rendering (powers glow)
- **Install:** `go get github.com/charmbracelet/glamour`
- **Type:** Go library only
- **Note:** Use `glow` CLI or `gum format` from bash instead.

### bubbletea — TUI framework

- **What:** Elm Architecture-based TUI framework for Go. Powers 18K+ apps.
- **Install:** `go get github.com/charmbracelet/bubbletea`
- **Type:** Go library only
- **Features:** Model-View-Update loop, mouse support, focus reporting, framerate renderer
- **Note:** Full-screen interactive TUIs. Would require rewriting Doey components in Go.

### bubbles — TUI component library

- **What:** Pre-built TUI components for Bubble Tea (text inputs, viewports, spinners, progress bars, lists, tables, file pickers)
- **Install:** `go get github.com/charmbracelet/bubbles`
- **Type:** Go library only
- **Note:** Components that power gum's interactive features. Not usable from bash.

### huh — Terminal forms and prompts

- **What:** Build interactive terminal forms with groups, validation, accessibility, and themes
- **Install:** `go get github.com/charmbracelet/huh`
- **Type:** Go library only
- **Note:** Multi-step form wizards. gum covers the bash-accessible subset of this.

### log — Colorful structured logging

- **What:** Minimal, colorful Go logging library with levels and structured fields
- **Install:** `go get github.com/charmbracelet/log`
- **Type:** Go library only
- **Note:** Exposed via `gum log` subcommand for bash usage.

### harmonica — Spring animation library

- **What:** Physics-based animation (spring, damping) for smooth terminal motion
- **Install:** `go get github.com/charmbracelet/harmonica`
- **Type:** Go library only

### sequin — ANSI sequence parser

- **What:** Human-readable ANSI sequence parsing and rendering
- **Install:** `go get github.com/charmbracelet/sequin`
- **Type:** Go library only

## Tier 3: Specialized CLI Tools (Not Relevant for Doey UX)

### soft-serve — Self-hostable Git server

- **What:** TUI-based git server over SSH/HTTP with repo management, hooks, LFS
- **Install:** `brew install soft-serve`
- **Relevance:** None for Doey — it's a git hosting server

### wish — SSH app framework

- **What:** Go library for building SSH-accessible TUI applications with middleware
- **Type:** Go library only
- **Relevance:** None — Doey is local, not SSH-served

### wishlist — SSH directory

- **What:** CLI for managing SSH endpoint connections
- **Relevance:** None for Doey

### crush — Agentic coding CLI

- **What:** Charmbracelet's AI coding agent (like Claude Code, built on their TUI stack)
- **Relevance:** Competitor/inspiration, not a dependency. Uses LSP, MCP, Agent Skills standard. Note: Mods (predecessor) was sunset March 2026, replaced by Crush.

### fantasy — AI agent SDK for Go

- **What:** Build AI agents with Go, supporting multiple LLM providers
- **Type:** Go library only
- **Relevance:** None for Doey unless building Go-based agents

### melt — SSH key backup

- **What:** Backup/restore Ed25519 SSH keys with seed words
- **Relevance:** None for Doey

### skate — Personal key-value store

- **What:** Local key-value store CLI (removed cloud sync in v1.0)
- **Install:** `brew install skate`
- **Relevance:** Possible — could store Doey config/state, but overkill vs flat files

|----------|------|--------|--------|
| **P0** | gum | Install as dependency, integrate into doey.sh | Transforms every user interaction — prompts, spinners, tables, styled output |
| **P1** | glow | Install as optional dep, use for help/docs | Rich markdown help rendering |

## Install Snippet (for Doey's install.sh)

```bash
# Check for gum, suggest install if missing
if ! command -v gum &>/dev/null; then
    echo "Optional: Install gum for enhanced UI"
    echo "  brew install gum"
    echo "  # or: go install github.com/charmbracelet/gum@latest"
fi

# Check for glow, suggest install if missing
if ! command -v glow &>/dev/null; then
    echo "Optional: Install glow for rich markdown display"
    echo "  brew install glow"
fi
```

## Key Insight

**gum is the entire Charmbracelet ecosystem packed into one bash-friendly binary.** It wraps lipgloss (via `gum style`/`gum join`), glamour (via `gum format`), huh (via `gum input`/`gum choose`/`gum confirm`), bubbles spinners (via `gum spin`), and log (via `gum log`). For a bash-based project like Doey, gum is the only must-have. Everything else is either Go-only or niche tooling.
