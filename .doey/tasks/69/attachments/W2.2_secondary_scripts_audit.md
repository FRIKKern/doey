## Key Observations for Charmbracelet Upgrade

1. **Three distinct color constant naming conventions** exist: `BRAND/SUCCESS/...` (install.sh, pre-push-gate.sh), `C_DIM/C_CYAN/...` (info-panel.sh, settings-panel.sh), and `WARN/ERROR/...` (context-audit.sh). These should be unified.

2. **`info-panel.sh` and `settings-panel.sh`** are the two heaviest TUI scripts (~510 and ~1265 lines). Both implement their own screen clearing, cursor management, layout, and refresh loops. These are the primary candidates for charmbracelet/lipgloss replacement.

3. **`install.sh`** has the most elaborate step-progress system (7 steps with `вң“`/`вң—` feedback). This is a strong candidate for a charmbracelet spinner/progress component.

4. **`web-install.sh`** uses NO colors at all вҖ” just plain echo with Unicode symbols. Lowest priority for upgrade.

5. **`doey-render-task.sh`** already has an ASCII fallback system (`DOEY_ASCII_ONLY`). Any upgrade should preserve this accessibility feature.

6. **`doey-tunnel.sh`** and **`doey-remote-provision.sh`** use log-format output вҖ” these may not need charmbracelet treatment (they run in background/remote contexts).

7. **Box drawing is manual** everywhere: `в”Ңв”Җв”җв”Ӯв””в”Җв”ҳ` in install.sh, `в•җв•җв•җ` and `в”Җв”Җв”Җ` in info-panel.sh. Lipgloss borders would replace all of these.

8. **Three files have interactive prompts**: `install.sh` (`read -r`), `settings-panel.sh` (`read -r` + `read -s -n 1`). These map to huh/bubbletea inputs.

9. **`pre-push-gate.sh`** has a well-structured checkвҶ’result pattern with dotted leaders вҖ” a natural fit for a charmbracelet table or list component.

10. **The `doey-statusline.sh` and `tmux-statusbar.sh`** output is consumed by tmux, not terminals directly. These use tmux-specific formatting and should probably remain as-is.
