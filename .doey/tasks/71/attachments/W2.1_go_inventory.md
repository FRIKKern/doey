## 2. Existing Build Infrastructure

### install.sh (lines 274-354)

**Go detection** — `_find_go()` function:
- Checks `command -v go` first
- Falls back to `/usr/local/go/bin/go`, `/opt/homebrew/bin/go`
- Sets `GO_BIN` variable (empty if not found)

**Build function** — `_build_tui()`:
1. Runs `go mod tidy` then `go build -o ~/.local/bin/doey-tui ./cmd/doey-tui/`
2. On success: also builds `doey-remote-setup` (non-fatal — failure is silent)
3. On failure: warns user, notes `info-panel.sh` fallback

**Go auto-install** — If Go is missing AND `go.mod` exists AND Homebrew available:
- Offers to `brew install go`
- Retries build after install

**Step label:** `[6/7] Installing doey-tui...`

### pre-commit-go.sh (git pre-commit hook)

- **Installed by:** `install.sh` lines 347-354 (copies to `.git/hooks/pre-commit`)
- **Trigger:** Only when `tui/` files are staged (`git diff --cached --name-only | grep '^tui/'`)
- **Action:** `cd tui && go build ./...`
- **On failure:** Blocks commit with error message
- **On missing Go:** Warns but does NOT block (exit 0)

### on-pre-tool-use.sh (Claude Code hook, lines 292-315)

- **Go build gate for Manager role** (blocks `git commit` commands)
- **Trigger:** When staged files include `tui/` AND tool command contains `git commit`
- **Go discovery:** Same pattern — `/snap/go/current/bin`, `/usr/local/go/bin`, `$HOME/go/bin`
- **Action:** `cd tui && go build ./...`
- **On failure:** Blocks the Bash tool call with `exit 2` and error output
- **On missing Go:** Silent skip (no warning, no block)

### pre-push-gate.sh (quality gate)

- **Detects language** from `go.mod` (checks root, then one subdir level)
- **Build check:** `cd $GO_MODULE_DIR && go build ./...`
- **Also runs:** `go test ./...`, `golangci-lint run` (if available)
- **Go discovery:** `/snap/go/current/bin`, `/usr/local/go/bin`, `$HOME/go/bin`, `$HOME/.local/go/bin`
- **Already has gum integration** — `HAS_GUM` detection and styled output

### post-tool-lint.sh (bash 3.2 lint)

- Only lints `.sh` files — does NOT touch `.go` files
- Not relevant to Go build pipeline

### doey doctor (doey.sh lines 2826-2834)

- **Checks:** `command -v doey-tui`
- **If found:** Reports version via `doey-tui --version`
- **If missing + Go available:** `warn "Go available — run install.sh to build"`
- **If missing + no Go:** `skip "using info-panel.sh fallback"`
- **Does NOT check:** Binary freshness, source file modification times, doey-remote-setup

### doey update/reinstall (doey.sh lines 2259-2360)

- Runs `install.sh` from updated code (via `_post_update` or direct call)
- **install.sh rebuilds doey-tui** as part of its normal flow (step 6/7)
- So `doey update` DOES rebuild Go binaries (indirectly, via install.sh)

### doey uninstall (doey.sh lines 2721-2737)

- Removes `~/.local/bin/doey`, `tmux-statusbar.sh`, `pane-border-status.sh`
- **Does NOT remove:** `doey-tui`, `doey-remote-setup` from `~/.local/bin/`
- **Gap:** Orphaned Go binaries survive uninstall

## 3. Gaps

### Missing from Build Pipeline

| Gap | Impact | Priority |
|-----|--------|----------|
| **No binary freshness check** — doctor/launch never compare binary mtime vs source mtime | Stale binary runs after `git pull` without `install.sh` | High |
| **No auto-rebuild on `doey` launch** — if Go source is newer than binary, nothing happens | Developer must manually run `install.sh` or rebuild | High |
| **doey-remote-setup not in doctor** — only doey-tui is checked | No visibility into remote-setup binary status | Low |
| **doey-remote-setup not in uninstall** — `uninstall_system()` doesn't clean it up | Orphaned binary | Medium |
| **No `doey build` command** — no way to rebuild Go binaries without re-running full install.sh | Developer friction for Go-only changes | Medium |
| **Version hardcoded** — `const version = "doey-tui v0.1.0"` not injected via ldflags | Version doesn't track git commits | Low |
| **Inconsistent Go discovery** — 4 different Go path search patterns across files | If one is wrong, behavior differs per context | Low |
| **pre-commit hook installed by install.sh only** — fresh clone + no install.sh = no hook | Contributors may commit broken Go code | Low |
| **doey reload doesn't rebuild** — reload restarts agents but skips Go binary rebuild | Changed Go code not picked up by reload | Medium |
| **No `go vet` in any gate** — pre-push-gate runs build/test/lint but not vet | Misses some Go issues | Low |

## 4. Recommendations

### R1: Unified Go discovery function (shared helper)

Create a `_find_go_bin()` function in a shared location (e.g., sourced from `shell/doey-go-helpers.sh` or inlined into each script) that checks ALL known paths:

```bash
_find_go_bin() {
  command -v go >/dev/null 2>&1 && { echo "go"; return 0; }
  for d in /usr/local/go/bin /opt/homebrew/bin /snap/go/current/bin \
           "$HOME/go/bin" "$HOME/.local/go/bin"; do
    [ -x "$d/go" ] && { echo "$d/go"; return 0; }
  done
  return 1
}
```

Use in: `install.sh`, `pre-commit-go.sh`, `on-pre-tool-use.sh`, `pre-push-gate.sh`.

### R2: Binary freshness check

Add to `doey doctor` and optionally to session launch:

```bash
_go_binary_stale() {
  local bin="$1" src_dir="$2"
  [ -f "$bin" ] || return 0  # missing = stale
  local bin_mtime src_mtime
  bin_mtime=$(stat -f '%m' "$bin" 2>/dev/null || stat -c '%Y' "$bin" 2>/dev/null) || return 1
  src_mtime=$(find "$src_dir" -name '*.go' -newer "$bin" 2>/dev/null | head -1)
  [ -n "$src_mtime" ]  # return 0 if any .go file is newer
}
```

### R3: `doey build` subcommand

Add a lightweight `build` subcommand to doey.sh that:
1. Finds Go binary
2. Runs `cd tui && go build -o ~/.local/bin/doey-tui ./cmd/doey-tui/`
3. Optionally builds `doey-remote-setup`
4. Reports success/failure

### R4: Auto-rebuild on launch (optional)

In `_init_doey_session()` or the session start path, check if `tui/` source is newer than `~/.local/bin/doey-tui` and offer to rebuild (or auto-rebuild silently).

### R5: Fix uninstall

Add to `uninstall_system()`:
```bash
rm -f ~/.local/bin/doey-tui ~/.local/bin/doey-remote-setup
```

### R6: Add doey-remote-setup to doctor

```bash
if command -v doey-remote-setup >/dev/null 2>&1; then
  _doc_check ok "doey-remote-setup" "installed"
else
  _doc_check skip "doey-remote-setup not installed" "(optional)"
fi
```

### R7: Version injection via ldflags (optional, low priority)

```bash
GIT_VERSION=$(git -C "$SCRIPT_DIR" rev-parse --short HEAD 2>/dev/null || echo "dev")
go build -ldflags "-X main.version=doey-tui-${GIT_VERSION}" -o ~/.local/bin/doey-tui ./cmd/doey-tui/
```

### R8: Add `doey reload` Go rebuild

In the reload function, add a step to check binary freshness and rebuild if Go source has changed:
```bash
if _go_binary_stale "$HOME/.local/bin/doey-tui" "$repo_dir/tui"; then
  printf "  ${DIM}Rebuilding doey-tui...${RESET}\n"
  _build_tui
fi
```
