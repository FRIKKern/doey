# Scaffy TUI Tab — Masterplan

## Goal

Build a dedicated **Scaffy** tab in the Doey Go TUI that lets users browse, run, and manage scaffy templates without leaving the interface. First-class experience for template discovery, creation, auditing, and execution.

## Status: 📋 Plan Ready — Awaiting User Review

---

## Executive Summary

Research reveals a clean integration path. Scaffy is already a well-structured Go library (35+ source files, 8 packages) with zero Bubble Tea dependencies. The TUI has a proven tab pattern — embed `SplitPaneModel`, wire 8 touch points in `root.go`, and import scaffy packages directly. No subprocess shelling needed for reads; fork only for filesystem mutations.

**Estimated scope:** ~4 new files, ~50 lines changed in root.go, 1 keybinding update.

---

## Research Findings

### 1. Scaffy is a Library, Not Just a CLI

The scaffy engine (`tui/internal/scaffy/`) is fully importable as Go packages:
- `dsl.ScanTemplates()` — list templates from `.doey/scaffy/templates/`
- `dsl.Parse()` — parse and validate template syntax
- `audit.AuditTemplate()` — run 6 health checks, get structured results
- `engine.Plan()` — dry-run execution against in-memory filesystem overlay
- `engine.Execute()` — real execution with filesystem mutations
- `discover.FindStructuralPatterns()` / `FindAccretionFiles()` / `FindRefactoringPatterns()` — pattern discovery

This means the TUI tab can import these directly instead of shelling out.

### 2. TUI Tab Pattern is Well-Established

Adding a tab requires:
1. Create `tui/internal/model/scaffy.go` (the model)
2. Wire 8 touch points in `root.go` (field, tab list, constructor, snapshot, keys, mouse, view, sizes, focus)
3. The `SplitPaneModel` base (used by Plans, Tasks, Files) provides list + detail viewport layout for free
4. `PanelNine` keybinding already exists in `keys.go` (bound to `9`) — just needs wiring

### 3. Reference Pattern: Plans Tab

`PlansModel` is the ideal structural reference:
- Embeds `SplitPaneModel` (list left, detail right)
- Has action buttons (Build, Create Tasks, Backlog) with mouse zones
- Renders markdown in detail panel via glamour
- Uses `planCardDelegate` for custom 2-line list items
- Data loads via `SetSnapshot()` from the runtime reader

### 4. Data Layer

Templates are discovered via filesystem scan (`dsl.ScanTemplates()`) — no database needed. Audit results are computed on-demand. The Scaffy tab can:
- Load template list on init/refresh via direct import
- Run audits lazily (on template select or explicit action)
- Cache parsed `TemplateSpec` to avoid re-parsing

## Architecture

```
tui/internal/model/scaffy.go          ← NEW: Main Bubble Tea model
tui/internal/model/scaffy_delegate.go  ← NEW: Custom list card delegate
tui/internal/model/root.go            ← MODIFIED: 8 touch points for tab wiring
tui/internal/keys/keys.go             ← MODIFIED: PanelNine help text → "scaffy"
```

### Data Flow

```
Init/Refresh
  └── dsl.ScanTemplates(templatesDir)  →  []RegistryEntry
        └── for selected entry:
              ├── dsl.Parse(fileContent)        →  *TemplateSpec (detail view)
              ├── audit.AuditTemplate(spec,...)  →  AuditResult (health badge)
              └── engine.Plan(spec, opts)        →  *PlanReport (dry-run preview)

Actions
  ├── 'r' Run   → exec.Command("doey-scaffy", "run", ...) (fork for safety)
  ├── 'a' Audit → audit.AuditTemplate() direct import
  ├── 'v' Validate → dsl.Parse() direct import
  ├── 'n' New   → exec.Command("doey-scaffy", "new", ...) (needs stdin)
  └── 'd' Discover → discover.Find*() direct imports
```

### Model Struct

```go
type ScaffyModel struct {

    // Data

}
```

### Wave 4: Actions — Audit + Validate (~100 lines)

**Files:** `tui/internal/model/scaffy.go` (extend)

### Wave 5: Actions — Run + Discover (~80 lines)

## Key Type Reference

### Template Registry Entry (`dsl/registry.go`)
```go
type RegistryEntry struct {
    Name, Path, Description, Domain, Version, Author, ParseError string
    Tags []string
}
```

### Template Spec (`dsl/types.go`)
```go
type TemplateSpec struct {
    Name, Description, Version, Author, Domain, Concept string
    Tags      []string
    Variables []Variable
    Operations []Operation
}
```

### Audit Result (`audit/report.go`)
```go
type AuditResult struct {
    Template string        // template name
    Path     string        // file path
    Checks   []CheckResult // 6 checks
    Status   string        // "healthy", "needs_update", "stale"
}
```

### 6 Audit Checks
1. `anchor_validity` — INSERT/REPLACE anchors exist in target files
2. `guard_freshness` — UNLESS guards not already triggered
3. `path_existence` — CREATE targets absent, INSERT targets present
4. `variable_alignment` — Variables have transforms and defaults
5. `pattern_activity` — Target files have recent git activity
6. `structural_consistency` — CREATE directories match extension set

## Risk Assessment

| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| Tab count increase breaks keybinding math | Low | Explicit `% 9` update; test tab cycling |
| Scaffy import creates circular dependency | None | `model/` → `scaffy/` is one-way; scaffy has zero model imports |
| Long-running audit/discover blocks TUI | Medium | Use `tea.Cmd` goroutines; show loading state |
| Templates dir doesn't exist | Low | Graceful empty state; suggest `scaffy init` |
| File watching for template changes | Medium | Defer; manual refresh via SetSnapshot is sufficient |
