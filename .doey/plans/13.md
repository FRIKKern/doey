# Task File Cleanup — High-Standard .task Files

## Goal
Clean up all 142 .task files to a high quality standard, sync with SQLite DB, and commit to git. The task files are the project's historical record — they should be clean, consistent, and useful.

## Current State (researched)
- 142 .task files in .doey/tasks/, 2.8MB total
- 22 status mismatches between files and DB
- 39 files with TASK_REPORT_N noise entries (e.g., "Worker 2.3 error with 0 tool calls, 0 files changed")
- 13 files with polluted decision logs (raw CLI args instead of clean entries)
- 3 files with duplicate TASK_SUBTASKS fields
- 2 schema versions (v3 = 61 files, rest have no version or v2)
- Many files with empty placeholder fields (TASK_TAGS=, TASK_HYPOTHESES=, etc.)
- Files currently untracked by git (.doey/ in .gitignore)

## Steps

- [ ] Step 1: Write a cleanup script (bash) that processes each .task file (complexity: medium)
  - Sync TASK_STATUS from DB (doey-ctl task get)
  - Strip empty fields (TASK_TAGS=, TASK_ACCEPTANCE_CRITERIA=, TASK_HYPOTHESES=, TASK_BLOCKERS=, TASK_RELATED_FILES=, TASK_NOTES= when empty)
  - Clean decision logs: remove raw CLI arg entries (lines starting with --type decision --author), deduplicate "Status changed to done" entries
  - Remove noise TASK_REPORT_N entries: drop reports where body matches "completed with N tool calls, 0 files changed" pattern
  - Keep meaningful reports (those with actual content from Managers, non-zero files changed, or substantive body text)
  - Deduplicate TASK_TIMESTAMPS (keep first created=, last completed=)
  - Remove duplicate fields (e.g., double TASK_SUBTASKS)
  - Normalize schema version to 3
  - Remove trailing status=done lines (artifact of doey-ctl appends)

- [ ] Step 2: Run the cleanup script on all 142 files (complexity: low)
  - Verify output on 5 sample files before bulk run
  - Back up originals to /tmp/doey/task-backup/ first

- [ ] Step 3: Update .gitignore to allow .doey/tasks/ and .doey/plans/ (complexity: low)
  - Add !.doey/tasks/ and !.doey/plans/ exceptions after .doey/ entry
  - Keep ignoring .doey/doey.db and other runtime state

- [ ] Step 4: Commit cleaned task files + plans to git (complexity: low)
  - Stage .doey/tasks/*.task and .doey/plans/*.md
  - Commit with descriptive message
  - Push to main

## Risks
- Script could corrupt task files: mitigated by backup + sample verification before bulk run
- .gitignore exception could expose DB or runtime files: mitigated by specific path exceptions only

## Scope
- **Files**: 142 .task files + .gitignore + cleanup script
- **Teams**: 1 worker team (scripting task)
- **Workers**: 2 (script writer + verifier)

## Acceptance Criteria
- Zero status mismatches between .task files and DB
- No TASK_REPORT entries with "0 files changed" noise
- No raw CLI args in decision logs
- No duplicate fields in any file
- No empty placeholder fields
- All files at schema version 3
- .doey/tasks/ and .doey/plans/ tracked in git
- .doey/doey.db and other runtime files NOT tracked
- Clean commit on main
