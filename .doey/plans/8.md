# TUI SQLite full integration — migrate remaining file reads to store

## Goal
Ensure the Go TUI fully leverages SQLite for all data the store supports. Currently 5 of 10 data types use store-backed reads with file fallback. Migrate the remaining gaps.

## Current State

### Already store-backed (with file fallback) ✓
1. Tasks (readTasks — includes subtasks + logs from store)
2. Pane statuses (readPaneStatuses)
3. Agents (readAgents)
4. Plans (readPlans)
5. Messages (readMessages)

### File-only — candidates for store migration
6. **Teams** — store has `teams` table + UpsertTeam/GetTeam/ListTeams, but TUI reads team_N.env files only
7. **Events** — store has `events` table + ListEvents, but TUI has no events display yet
8. **Config** — store has `config` table + Get/Set/List, but TUI has no config display

### File-only — should STAY file-only (runtime/ephemeral)
9. Session config (session.env) — runtime bootstrap, not DB material
10. Results (pane_*.json) — transient worker output
11. Context percentages — transient monitoring
12. Uptime — derived from session.env stat
13. Team definitions (*.team.md) — definition files, not state
14. Connections — external config
15. Debug entries — transient

### Subtask gap
- store_reader.go readTasks() already reads subtasks from store ✓
- BUT ReadSnapshot still calls parseSubtasks() from files separately and re-attaches them
- This means subtasks get read twice (from store inside readTasks, then overwritten from files)
- Fix: skip file-based parseSubtasks when store data was used

## Steps
- [ ] Step 1: Add `readTeams()` to store_reader.go — read teams table, convert to TeamConfig (medium)
- [ ] Step 2: Wire teams store read into ReadSnapshot with file fallback (low)
- [ ] Step 3: Fix subtask double-read — skip parseSubtasks() when store provided tasks with subtasks (low)
- [ ] Step 4: Add `readEvents()` to store_reader.go — expose events for future TUI tab (low)
- [ ] Step 5: Add `readConfig()` to store_reader.go — expose config for future TUI display (low)
- [ ] Step 6: go build + verify TUI renders correctly (low)

## Risks
- Teams table may not be populated yet (migration hasn't run): file fallback handles this
- Subtask fix could regress if store returns empty subtasks for tasks that have file-based ones: check length before skipping

## Scope
- **Files**: tui/internal/runtime/store_reader.go, tui/internal/runtime/reader.go, tui/internal/runtime/types.go
- **Teams**: 1 team, 2 workers
- **Workers**: 2

## Acceptance Criteria
- store_reader.go has readTeams(), readEvents(), readConfig() methods
- ReadSnapshot uses store teams with file fallback
- Subtasks not double-read when store provides them
- go build ./... passes
- TUI renders tasks, plans, teams, statuses correctly
