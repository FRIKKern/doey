# Taskmaster Rename — Task/Subtask Obsessed System

## Goal
Rename Session Manager → "Taskmaster" and Window Manager → "Subtaskmaster" everywhere. Ban all abbreviations (SM, TM, WM). Update CLAUDE.md to codify naming and task-obsessed philosophy. Real-time subtask tracking is the core feature.

## Scope
- **48 files** across agents, hooks, skills, shell, Go TUI, docs, CLAUDE.md
- **~200+ references** to "Session Manager", "Window Manager", "SM", "WM"
- **4 files/dirs** need renaming (agent defs, hook, skill dir)
- **Variable names**: SM_SAFE, SM_PANE, _sm_ prefixes in shell code

## Phases

### Phase 1: Agents + CLAUDE.md (foundation)
- Rename agents/doey-session-manager.md → agents/doey-taskmaster.md
- Rename agents/doey-window-manager.md → agents/doey-subtaskmaster.md
- Update content in both + all agent files referencing SM/WM
- Update CLAUDE.md: rename roles, add naming rule, add task-obsessed philosophy

### Phase 2: Hooks (high risk — runtime)
- Rename session-manager-wait.sh → taskmaster-wait.sh
- Update all hooks: common.sh, on-notification.sh, on-pre-compact.sh, on-pre-tool-use.sh, on-session-start.sh, stop-notify.sh
- Rename SM_SAFE/SM_PANE/_sm_ variables → taskmaster equivalents
- Update doey.sh hook registration

### Phase 3: Skills
- Rename doey-sm-compact/ → doey-taskmaster-compact/
- Update all 14 skill files referencing SM/WM

### Phase 4: Shell scripts (high risk — core runtime)
- doey.sh (19 refs + variable names)
- doey-ipc-helpers.sh, doey-config-default.sh, info-panel.sh, pane-border-status.sh

### Phase 5: Go TUI
- model/actions.go, plans.go, welcome.go
- grammar/types.go, parser_test.go, terminal_test.go
- runtime/reader.go
- go build ./... to verify

### Phase 6: Docs + verification
- docs/context-reference.md, configuration.md, research/
- tests/test-bash-compat.sh
- Final grep for any remaining SM/WM references

## Risks
- Hook file rename breaks runtime (doey.sh registers by filename)
- Variable rename breaks IPC message routing (SM_SAFE used for addressing)
- Skill rename changes slash command name (/doey-sm-compact → /doey-taskmaster-compact)

## Acceptance Criteria
- Zero "Session Manager", "Window Manager", "SM" (as abbrev), "WM" in codebase
- CLAUDE.md has naming convention + task-obsessed philosophy
- go build passes, bash compat tests pass
- Hooks load, message routing works
