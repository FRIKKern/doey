# Go Event Router

## Goal
Replace Taskmaster's token-heavy mechanical work (message reading, status polling, routing) with a Go daemon. Taskmaster only wakes for judgment calls — conflicts, errors, cross-team decisions.

## Architecture

Subtaskmasters write .msg files. doey-router (Go) watches triggers, reads msgs, classifies them. Routine = handle in Go + log. Judgment = wake Taskmaster with pre-digested context.

## Message Classification

ROUTINE (Go handles, no Taskmaster wake):
- task_complete: update .task status, log, notify Boss
- status_report: log only
- phase_complete: dispatch next phase from plan
- health_ok: no action

JUDGMENT (wake Taskmaster):
- error / blocked: needs decision
- question from Subtaskmaster: needs answer
- cross-team conflict: needs coordination
- ambiguous / unknown subject: safe fallback

## Steps

- [ ] Step 1: Research — audit all msg types Taskmaster handles, classify routine vs judgment (low)
- [ ] Step 2: Core router — tui/cmd/doey-router/main.go with trigger watcher, msg parser, classifier (high)
- [ ] Step 3: Routine handlers — auto-routing for task_complete, phase_complete, status updates (medium)
- [ ] Step 4: Taskmaster wake — compose context + send-keys when judgment needed (medium)
- [ ] Step 5: Structured trace log — JSON event log at RUNTIME_DIR/trace.jsonl (low)
- [ ] Step 6: Session startup — launch doey-router as background process in doey.sh (low)
- [ ] Step 7: Verification — full task lifecycle with router, compare token usage (medium)

## Risks
- Router crash = Taskmaster never wakes. Mitigation: watchdog, fallback to direct msg reading
- Misclassification. Mitigation: conservative, unknown = always wake
- Race conditions. Mitigation: serial processing per trigger batch
- taskmaster-wait.sh conflict. Mitigation: router replaces when active, coexists when not

## Scope
- Files: new tui/cmd/doey-router/, shell/doey.sh, taskmaster-wait.sh
- Teams: 1 team, 2-3 workers

## Acceptance Criteria
- Router starts automatically with session
- Routine messages handled without waking Taskmaster
- Judgment messages wake Taskmaster with pre-digested context
- Structured trace log captures all events
- Graceful fallback if router not running
- Measurable token reduction
