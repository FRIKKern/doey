# Task Completion Pipeline — Review Gate + Deployment Separation

## Goal
Redesign the task completion flow so that:
1. Task Reviewer MUST update .task files with review proof before any commit happens
2. Git commit/push is ONLY done by the Deployment specialist (pane 1.2)
3. Task Reviewer notifies Deployment after review passes
4. Workers and Subtaskmasters are fully blocked from git commit/push

## Current Flow (what's wrong)
```
Worker finishes → stop hooks → Subtaskmaster consolidates
→ Taskmaster receives task_complete → Taskmaster does git commit/push
→ Boss notified
```
Problems:
- Taskmaster does git commit — should be Deployment's job
- Task Reviewer is wired but passive — doesn't update task files with proof
- Subtaskmasters (TEAM_LEAD role) are NOT blocked from git commit/push in on-pre-tool-use.sh
- No gate between "work done" and "commit" — review is advisory, not mandatory

## New Flow
```
Worker finishes → stop hooks → Subtaskmaster consolidates
→ Subtaskmaster sends task_complete to Taskmaster
→ Taskmaster routes review request to Task Reviewer (1.1)
→ Task Reviewer reads code + task, writes REVIEW verdict to .task file
→ On PASS: Task Reviewer sends deployment_request to Deployment (1.2)
→ On FAIL: Task Reviewer sends review_failed to Taskmaster with fix list
→ Deployment runs pre-deploy checks, commits, pushes
→ Deployment sends deployment_complete to Taskmaster
→ Taskmaster reports to Boss
```

## Steps

- [ ] Step 1: Block git commit/push for Subtaskmasters AND Taskmaster in on-pre-tool-use.sh (complexity: low)
  - The git block currently only applies to roles != coordinator && != deployment
  - Change: remove coordinator from the exemption — only deployment is allowed git ops
  - Subtaskmasters (TEAM_LEAD) will be caught by this same change
  - File: `.claude/hooks/on-pre-tool-use.sh` (~line 375)

- [ ] Step 2: Update Task Reviewer agent to write review findings to .task file (complexity: medium)
  - Current agent outputs a text verdict but doesn't update the task file
  - Must: read task → review code → write TASK_REVIEW_* fields to .task file via doey-ctl
  - Fields: TASK_REVIEW_VERDICT (PASS/FAIL), TASK_REVIEW_FINDINGS, TASK_REVIEW_TIMESTAMP
  - On PASS: send deployment_request message to Deployment (1.2) via doey-ctl msg send
  - On FAIL: send review_failed message to Taskmaster with specific fixes needed
  - Task Reviewer already has Edit/Write permission on .doey/tasks/*
  - File: `agents/doey-task-reviewer.md.tmpl`

- [ ] Step 3: Update Deployment agent to handle commit/push workflow (complexity: medium)
  - Current agent is generic "run tests, push, create PR"
  - Must: receive deployment_request from Task Reviewer → verify review passed in .task → run pre-deploy checks → git add/commit/push → send deployment_complete to Taskmaster
  - Deployment already has git push + gh pr create allowed in on-pre-tool-use.sh
  - Must also ensure git commit is allowed for Deployment (verify in hook)
  - File: `agents/doey-deployment.md.tmpl`

- [ ] Step 4: Update Taskmaster to route completions through Task Reviewer (complexity: medium)
  - Current: Taskmaster receives task_complete → commits directly
  - New: Taskmaster receives task_complete → sends review_request to Task Reviewer (1.1)
  - Taskmaster receives deployment_complete from Deployment → reports to Boss
  - Taskmaster receives review_failed → routes fix request back to team
  - File: `agents/doey-taskmaster.md.tmpl`

- [ ] Step 5: Expand templates and verify (complexity: low)
  - Run bash shell/expand-templates.sh to regenerate .md from .md.tmpl
  - Run tests/test-bash-compat.sh for hook changes
  - Verify on-pre-tool-use.sh blocks correctly

## Risks
- Taskmaster currently does commits: block git ops for Coordinator role in hook
- Message delivery: use doey-ctl msg send (already works for Core Team)
- Task Reviewer send-keys blocked: fine — use doey-ctl msg send instead
- Circular: Deployment failure → reports to Taskmaster → Taskmaster re-routes

## Scope
- Files: on-pre-tool-use.sh, doey-task-reviewer.md.tmpl, doey-deployment.md.tmpl, doey-taskmaster.md.tmpl
- Teams: 1 team (4 files, focused changes)
- Workers: 2-3

## Acceptance Criteria
- Subtaskmasters + Taskmaster blocked from git commit/push (only Deployment allowed)
- Task Reviewer writes TASK_REVIEW_* fields to .task before any commit
- Deployment is the sole committer — receives request from Task Reviewer
- Review FAIL routes back to Taskmaster for re-dispatch
- tests/test-bash-compat.sh passes
- Templates expanded cleanly
