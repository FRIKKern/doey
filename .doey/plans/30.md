# Masterplan Dedicated Window

## Goal
When /doey-masterplan runs, spawn a dedicated tmux window with a Cursor-inspired layout:
- Window NAME = masterplan title (visible in tmux tab bar)
- LEFT pane (tall): Masterplan Agent (Claude planner)
- TOP-RIGHT pane (large): Live plan document viewer — scrollable, auto-refreshes
- BOTTOM-RIGHT: 4 worker panes in a horizontal row

```
┌──────────────┬─────────────────────────────┐
│              │                             │
│              │      Masterplan (live)      │
│   Planner    │      scrollable viewer      │
│   (Claude)   │                             │
│              ├──────┬──────┬──────┬────────┤
│              │  W1  │  W2  │  W3  │   W4   │
└──────────────┴──────┴──────┴──────┴────────┘
```

User screenshot reference: .doey/tasks/task-attachments/masterplan-cursor-inspiration.png

## Steps

- [ ] **Step 1: Plan viewer shell script — shell/masterplan-viewer.sh** (complexity: medium)
  A shell process (like info-panel.sh) that:
  - Watches a .md plan file for changes (poll every 1-2s, or inotifywait/fswatch if available)
  - Renders markdown with `glow` (if available) or `bat --style=plain` or plain `cat`
  - Clears and re-renders on file change
  - Supports terminal scrolling (user can scroll up/down in the tmux pane)
  - Shows plan title, status, phase progress at the top
  - Takes plan file path as argument: `masterplan-viewer.sh /path/to/plan.md`
  - Bash 3.2 compatible

- [ ] **Step 2: Masterplan team definition — teams/masterplan.team.md** (complexity: low)
  New team definition with custom layout:
  ```yaml
  name: masterplan
  description: "Masterplan window — planner + live plan viewer + 4 workers"
  grid: masterplan  # special layout type
  workers: 4
  type: local
  manager_model: opus
  panes:
    0: { role: planner, agent: doey-subtaskmaster, name: "Planner" }
    viewer: { role: viewer, script: masterplan-viewer.sh, name: "Plan" }
    1-4: { role: worker, name: "W{n}" }
  ```

- [ ] **Step 3: Masterplan layout builder in doey-team-mgmt.sh** (complexity: high)
  New layout type "masterplan" in the team window spawning code:
  - Create window with planner pane (full height left, ~35% width)
  - Split right side: top = viewer pane (~65% height), bottom = worker strip
  - Split worker strip into 4 equal horizontal panes
  - Planner pane: launch Claude with subtaskmaster agent
  - Viewer pane: launch `masterplan-viewer.sh <plan-file>`
  - Worker panes: launch Claude workers (initially idle/reserved)
  - Set window name to plan title: `tmux rename-window`
  - Register all panes in runtime env

- [ ] **Step 4: Update /doey-masterplan skill to spawn the window** (complexity: medium)
  Modify `.claude/skills/doey-masterplan/SKILL.md.tmpl`:
  - After intent/research phases, instead of dispatching to generic teams, run:
    `doey add-team masterplan --name "Plan: <title>" --task-id <N>`
  - Planner agent receives the plan file path and manages it
  - Workers get dispatched as phases are ready
  - Plan file path passed to viewer via env var or argument

- [ ] **Step 5: Planner → Viewer integration** (complexity: low)
  - Planner writes to `.doey/plans/<N>.md` using standard Write tool
  - Viewer watches that file and re-renders automatically
  - No IPC needed — file system is the communication channel
  - Planner can also write phase status updates that viewer renders as progress bars

## Risks
- **tmux layout complexity**: The 3-zone layout (left tall + top-right + bottom-right strip) requires careful split ordering. Mitigate: test exact split sequence in isolation first
- **Viewer performance**: Rapid file changes could cause flicker. Mitigate: debounce renders (minimum 0.5s between redraws)
- **glow/bat availability**: Not all systems have markdown renderers. Mitigate: graceful fallback to plain cat with ANSI color headers
- **Pane numbering**: Custom layout means pane indices won't match standard team pattern. Mitigate: explicit pane registration in team env

## Scope
- **Files**: `shell/masterplan-viewer.sh` (new), `teams/masterplan.team.md` (new), `shell/doey-team-mgmt.sh` (layout builder), `.claude/skills/doey-masterplan/SKILL.md.tmpl` (skill update)
- **Teams**: 1 team (2-3 workers)
- **Workers**: 2-3

## Acceptance Criteria
- `/doey-masterplan "goal"` spawns a window with the 3-zone layout matching the mockup
- Window name shows the plan title in tmux tab bar
- Plan viewer shows live markdown, updates automatically when planner writes
- User can scroll the plan viewer pane independently
- 4 worker panes appear in bottom-right strip, receive work from planner
- Layout survives terminal resize gracefully
