# Masterplan: doey-term Native Terminal Parity

**Task:** #380 | **Status:** ✅ Research Complete — Plan Ready

## Goal

Make doey-term behave like a real terminal emulator. Every keystroke that works in iTerm2, Alacritty, or macOS Terminal must work in doey-term.

## Architecture Summary

```
Bubble Tea v2 → term.Model (tabs) → bubbleterm → x/vt Emulator → creack/pty → shell
```

The VT emulator (`charmbracelet/x/vt`) and PTY (`creack/pty`) are solid. **The bottleneck is `taigrr/bubbleterm/keys.go`** — it's the choke point where all input is converted to terminal bytes, and it drops most of them.

## Root Cause Analysis

| Bug | Location | Cause |
|-----|----------|-------|
| **Space broken** | `bubbleterm/keys.go:22` | Matches `" "` but Bubble Tea v2 returns `"space"` from `msg.String()` |
| **Unicode broken** | `bubbleterm/keys.go:77` | `len(str) == 1` uses byte length — drops all multi-byte UTF-8 chars (ø, æ, å, emoji) |
| **Most Ctrl+key dropped** | `bubbleterm/keys.go` | Only 4 of 26 Ctrl combos mapped (c, d, z, l) — missing a, b, e, f, k, n, p, r, s, u, w, etc. |
| **Alt+key dropped** | `bubbleterm/keys.go` | No Alt+letter handling at all — should send `\x1b` + letter |
| **Delete wrong** | `bubbleterm/keys.go:18` | Sends `\x7f` (backspace) instead of `\x1b[3~` |
| **Width off by 2** | `bubbleterm/messages.go:65` | `emu.Resize(width-2, height)` — PTY always 2 columns narrow |
| **No cursor** | `bubbleterm/bubbleterm.go` | Cursor position from VT emulator never rendered in View() |
| **No scrollback UI** | `term/model.go` | VT stores 10k lines but no scroll-up keybinding or viewport |
| **33ms latency floor** | `term/model.go:272` | Tick-based polling instead of event-driven updates |

## Phased Plan

### Phase 1: Fix Critical Input Bugs (Day 1)
**Priority: CRITICAL — space and basic typing broken**

#### 1a. Fix space key
- **File:** `bubbleterm/keys.go:22`
- **Change:** `case " ":` → `case "space":`
- **Root cause:** Bubble Tea v2 API change — `Key.String()` returns `"space"` not `" "`

#### 1b. Fix unicode/international characters
- **File:** `bubbleterm/keys.go:76-81` (default case)
- **Change:** Replace `len(str) == 1` with `msg.Key().Text` check:
  ```go
  default:
      if text := msg.Key().Text; text != "" {
          return text  // Handles space, unicode, and all printable chars
      }
      return ""
  ```
- **Root cause:** `len()` returns byte count, not rune count. Using `Key().Text` bypasses the issue entirely and is the Bubble Tea v2 idiomatic approach.

#### 1c. Fix delete key
- **File:** `bubbleterm/keys.go:18`
- **Change:** Delete case should return `\x1b[3~` instead of `\x7f`

#### Verification
- Type: `hello world` (space works)
- Type: `echo ø æ å ü ñ` (unicode works)
- Press Delete key in vim (correct behavior)

### Phase 2: Complete Keyboard Coverage (Day 1-2)
**Priority: HIGH — shell editing broken without Ctrl keys**

#### 2a. Add all Ctrl+letter combos
- **File:** `bubbleterm/keys.go`
- **Change:** Add generic handler: if key string matches `ctrl+X`, send byte `X - 'a' + 1`
  ```go
  // Handle all ctrl+letter combinations generically
  if strings.HasPrefix(str, "ctrl+") && len(str) == 6 {
      ch := str[5]
      if ch >= 'a' && ch <= 'z' {
          return string([]byte{ch - 'a' + 1})
      }
  }
  ```
- **Critical combos:** Ctrl+A (0x01, line start), Ctrl+E (0x05, line end), Ctrl+R (0x12, reverse search), Ctrl+U (0x15, clear line), Ctrl+K (0x0B, kill to end), Ctrl+W (0x17, delete word)

#### 2b. Add Alt+key combos
- **File:** `bubbleterm/keys.go`
- **Change:** If key string matches `alt+X`, send `\x1b` + letter
- **Note:** Must coordinate with `term/model.go` which intercepts `alt+1..9`, `alt+[`, `alt+]` for tab management

#### 2c. Add Shift+arrow and Ctrl+arrow
- **File:** `bubbleterm/keys.go`
- **Change:** Add modifier-aware arrow key sequences (e.g., `\x1b[1;5A` for Ctrl+Up)

#### Verification
- In bash: Ctrl+A (cursor to start), Ctrl+E (cursor to end), Ctrl+R (reverse search), Ctrl+U (clear line)
- In vim: all Ctrl combos work as expected
- Alt+B (word back), Alt+F (word forward) in bash

### Phase 3: Rendering Fixes (Day 2-3)
**Priority: MEDIUM — visual correctness**

#### 3a. Fix width-2 offset
- **File:** `bubbleterm/messages.go:65`
- **Change:** `emu.Resize(width-2, height)` → `emu.Resize(width, height)` (or determine if borders exist that justify -2)
- **Impact:** Fixes line wrapping misalignment

#### 3b. Add cursor rendering
- **File:** `bubbleterm/bubbleterm.go` View() method
- **Change:** Query `emulator.CursorPosition()` and render a visible cursor at that position
- **Approach:** Use ANSI cursor positioning in the rendered output, or set cursor position in `tea.View`

#### 3c. Switch to event-driven updates
- **File:** `term/model.go` and `bubbleterm/bubbleterm.go`
- **Change:** Replace 33ms tick polling with PTY-read-triggered messages
- **Pattern:** PTY goroutine sends `tea.Msg` via `p.Send()` when data arrives, instead of polling every 33ms
- **Impact:** Eliminates latency floor, reduces CPU usage

#### Verification
- `echo "long line that wraps"` wraps at correct column
- Cursor visible and positioned correctly in vim
- Keystroke echo feels instant (no 33ms delay)

## Strategic Decision: Fix bubbleterm vs Replace It

**Phase 1-2: Fix bubbleterm** — Fastest path to working input. All changes are in `keys.go` (one file).

**Phase 3+: Consider replacing bubbleterm with custom x/vt wrapper** — TUIOS (github.com/Gaurav-Gosain/tuios) demonstrates the pattern. Benefits:
- Event-driven rendering (no polling)
- Direct access to x/vt damage tracking
- Cursor rendering via VT emulator API
- Full control over all input/output paths
- No upstream dependency risk

**Recommendation:** Fix `keys.go` first (Phase 1-2) for immediate relief, then evaluate whether Phase 3+ changes warrant a custom wrapper. The `x/vt` library is already a dependency — the integration pattern from TUIOS is proven.
