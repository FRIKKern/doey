# Plan: Improving the Plan pane inside the Masterplan Window

## Goal
Turn the Masterplan Plan pane (`doey-masterplan-tui`) into the most information-rich, Doey-native live planning view in the industry — surpassing Claude Code's native planning and Cursor's planning view — while preserving the existing interactive controls and surviving fresh installs unchanged.

## Context

**Source signal:** user request via `/doey-masterplan` to Boss on 2026-04-26 (`goal.md`). Deep Interview Brief at `.doey/plans/masterplan-20260426-203854.brief.md` is the synthesised requirement.

**User-confirmed v1 scope (planning-phase amendments to the brief):**
- Path: extend the existing `tui/cmd/doey-masterplan-tui/` binary in place (no parallel `doey-plan-view` binary).
- Go-missing path: install fails loud with platform-specific remediation hint (no prebuilt fallback).
- Information set: brief's three pillars (live plan markdown, consensus state header, research findings index) PLUS three additions explicitly chosen by the user — task wiring footer, reviewer last-vote summary, worker activity ticker. Plan diff highlights and expanded mouse/key bindings remain v2.

**W1 audit findings (`research/w1.md`):** the existing 820 LOC binary has no live reload at all (no `fsnotify`, no `tea.Tick`, no SIGHUP). Plan + consensus loaded once at startup. Half the parsed data model is invisible (`Goal`, `Context`, `Deliverables`, `Risks`, `SuccessCriteria`, `Phase.Body`). Mouse hit columns hardcoded. Three legacy flags accepted but discarded (`--runtime-dir`, `--goal`, `--team-window`). Consensus gate uses startup-cached value. Path drift between runtime convention (`/tmp/doey/<proj>/masterplan-*/`) and project convention (`.doey/plans/`) breaks the standalone fallback. `APPROVED` ↔ `CONSENSUS` aliasing inconsistency between badge and gate. `StatusFailed` non-recoverable through the TUI today. `expandedPhases` keyed by index (reorder bug).

**W2 signal catalog (`research/w2.md`):** rich Doey-native sources already exist — per-pane status files (STATUS, ACTIVITY, SINCE, LAST_ACTIVITY), heartbeat files for stall detection, `<PANE_SAFE>.role` for auto-discovery of Architect/Critic, `consensus.state` schema, reviewer verdict files, `${RUNTIME_DIR}/lifecycle/{events,alerts}.jsonl`, SQLite `plans` + `tasks` tables, `team_<W>.env` for plan binding. Existing `tui/internal/styles/` provides `TimelineKindColor/Icon/Label`, `StatusAccentColor`, `RenderListFrame`, `Theme.Default()`. `glamour` markdown renderer + `bubbles/{list,viewport,key}` + `lipgloss/v2` + `bubblezone` already on `go.mod`.

**Refresh model (decided):** hybrid `fsnotify` on plan markdown, `consensus.state`, verdict files, `research/` directory + per-file watches on first appearance, and per-pane `<PANE_SAFE>.status` for the Planner / Architect / Critic / W1 / W2; plus a 1s `tea.Tick` for clock/age/stall. On watcher error or `ENOSPC`: degrade to 1s polling and surface `WATCH: degraded` flag in footer. Self-write feedback loop suppressed via 200 ms persist-grace window.

**Atomic-rename rendezvous (decided):** every reader of `consensus.state`, verdict files, plan markdown, research files waits for `size-stable-for-100ms` before parsing, after first detecting a CHANGE/CREATE event. Rule applies to files that previously had content; legitimate 0-byte states during fresh-init are tracked separately (file-presence-without-content is a valid state). Editor rename-on-save handled by re-watching on `IN_MOVE_SELF`/inode change.

**Hard invariants:** fresh-install survives (`./install.sh && doey doctor` passes after `doey uninstall`); Bash 3.2-compatible shell glue; no edits to `~/.claude/settings.json`; truecolor + 256-color verified at 80/120/200-col widths.

**Explicit non-goals (defer to v2 or later):** `messages/` + `triggers/` queue surfacing, `lifecycle/events.jsonl` timeline, `lifecycle/alerts.jsonl` *deep* drilldown (a single banner is in v1; deeper inspector is v2), plan-diff highlights, expanded mouse/key bindings beyond what reviewer-cards / research-index need. Planner / Researcher / Boss / Taskmaster pane redesigns. Action-request hook fix.

**Dependency lock-in (decided up front):**
- Bubbletea + lipgloss: gate the new binary fully on **v2** (`charm.land/bubbletea/v2`, `charm.land/lipgloss/v2`). Verify nothing in this binary's transitive deps pulls v1. Document in `tui/cmd/doey-masterplan-tui/DECISIONS.md`. Cross-module migration of v1 callers (`tui/internal/model/plans.go`, etc.) is out of scope for v1 — adapter shim if any imports cross.
- Mouse hit-test: commit to **`bubblezone`** (already imported, used by `RenderButton`). Drop the hand-rolled `zone` struct from the draft entirely.

## Risks
- **Refresh thrash on noisy fsnotify** — mitigated by 100 ms per-file debounce + size-stable-for-100ms rendezvous + per-tick coalescing + idle-CPU < 1% acceptance test
- **`fs.inotify.max_user_watches` exhaustion** under heavy Doey sessions — distinguished `ENOSPC` triggers degraded polling with explicit footer flag; default budget ~10 watches per pane is well under 8192 limit per user
- **Self-write feedback loop** on persist — 200 ms persist-grace window suppresses the echo; cursor preserved by phase-identity hash (Phase 1)
- **Editor rename-on-save** breaks fsnotify by inode — `IN_MOVE_SELF` re-watch + explicit test
- **Mouse hit-test reflow** — eliminated by full bubblezone migration; no inline column literals; lint enforces
- **Hidden-section reveal blows vertical real-estate at 80 cols** — collapsible by default at <120, expanded ≥120, `o` overlay for full read
- **Install fails loud on Go-less box** — by user decision; mitigated by full platform fallback chain (brew/apt/dnf/pacman/apk/zypper/nix + generic go.dev/dl link); `doey doctor` mirrors the same hint
- **Verdict-format drift** — both forms accepted in shell + Go reader, one canonical form pinned in agent templates; regression test against existing in-flight masterplans
- **Path unification regressions** — every entry path has a unit test; standalone fallback explicitly UI'd; rollback flag (`--legacy`) provides a single-release escape hatch
- **Watcher reliability on macOS / network mounts** — degraded-mode polling + footer flag
- **Bubbletea v1↔v2 type incompatibility** — binary fully on v2, verified via `go mod why`; v1 callers in `tui/internal/model/plans.go` etc. remain on v1 (out of scope); adapter shim only if a transitive import crosses
- **Regression-harness maintenance tax** — smoke gate is single-fixture/single-width/single-profile; wider matrix opt-in (`make test-render-matrix`); `make refresh-render-goldens` for dependency bumps
- **`<PANE_SAFE>.role` not yet populated** on a brand-new window — falls back to pane index `.2`/`.3` with on-screen `role-via-index` indicator
- **Multiple parallel masterplans** — all pane lookups scoped by `DOEY_TEAM_WINDOW` (Phase 2 watcher contract)
- **Demo mode writes to live** — `Source == Demo` short-circuits every write path (Phase 4 acceptance)
- **glamour/lipgloss version bumps regenerate goldens en masse** — documented refresh command, opt-in matrix avoids CI gate
