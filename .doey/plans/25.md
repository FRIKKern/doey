# Masterplan: Fix Plans Tab Permanently

## Intent
After 14+ failed fix attempts, the Plans tab viewport/scrolling issues persist because patches targeted symptoms rather than the structural root cause. The true fix requires: (1) correcting the viewport height math to match Tasks, and (2) extracting a shared component so they can't diverge again.

## Phase 1: Fix Viewport Height + API Misuse

### What it delivers
Plans tab scrolls, clicks, and navigates identically to Tasks tab. Viewport starts at top. Works on small terminals.

### Changes
- plans.go:211 — change h-7 to h-3 (viewport height)
- plans.go:932 — change h-7 to h-3 (viewport height)
- plans.go:994-997 — remove manual line-clipping hack
- plans.go:409, :454, :459, :768 — replace YOffset=0 with GotoTop()

### Verification Gate
- Plans viewport starts at top, not bottom
- Scroll up/down works with keyboard and mouse wheel
- Tab switching works — no mouse traps
- Works on small terminal (< 40 rows)
- Tasks tab still works (no regression)
- Go build passes

## Phase 2: Structural Unification — Shared SplitPaneModel

### What it delivers

### Changes
- Extract shared SplitPaneModel from tasks.go
- Refactor plans.go to use shared SplitPaneModel
- Verify tasks.go still works with shared component

### Verification Gate
- Both tabs behave identically
- Both use same SplitPaneModel (no duplicated viewport logic)
- Go build passes
- Scroll, mouse, resize all work on both tabs
