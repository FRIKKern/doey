# Masterplan: Task Creation + Live Plan Viewer

## Goal
1. /doey-masterplan should auto-create a task when spawning — the plan IS a task
2. Masterplan pane 1 should use the Go TUI (doey-tui) instead of masterplan-viewer.sh, so the plan renders live in the same TUI used in the Dashboard

## Steps

- [ ] Step 1: Update /doey-masterplan skill to create a task on spawn (complexity: low)
  - After creating the plan working dir, run doey task create with the masterplan goal
  - Link task to plan via plan_id field
  - Pass TASK_ID to the Planner via the goal.md or masterplan.env
  - Planner updates the task as work progresses

- [ ] Step 2: Replace masterplan-viewer.sh with doey-tui in pane 1 (complexity: medium)
  - In teams/masterplan.team.md, change pane 1 from script: masterplan-viewer.sh to launching doey-tui
  - doey-tui takes runtime-dir as arg, already has a Plans tab
  - Need to verify plans written to .doey/plans/ appear in the TUI (DB sync path)
  - May need doey-tui to accept --tab=plans flag to default to plans view

- [ ] Step 3: Ensure plan file writes sync to DB for TUI display (complexity: medium)
  - The planner writes to .doey/plans/<id>.md
  - doey-ctl plan commands use DB
  - Need to verify the daemon or TUI reads from files, or add file->DB sync
  - If plans aren't synced: planner should use doey-ctl plan create/update instead of raw file writes

- [ ] Step 4: Clean up masterplan-viewer.sh if replaced (complexity: low)

## Risks
- doey-tui may not show plans from file system (only DB) — need to check sync path
- doey-tui in a small pane might not render well — needs testing

## Scope
- Files: /doey-masterplan skill, teams/masterplan.team.md, possibly doey-tui Go code
- Workers: 2

## Acceptance Criteria
- /doey-masterplan creates a tracked task on spawn
- Pane 1 shows Go TUI with live plan rendering
- Plan updates from Planner appear in real-time in pane 1
