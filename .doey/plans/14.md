# Systemic Reliability Fixes — Session Learnings

## Goal
Fix 7 systemic issues identified during the current session that cause worker blocks, Taskmaster stalls, communication failures, and poor observability.

## Steps

### Wave 1 — Critical (blocks work)
- [ ] Step 1: Taskmaster auto-compact and auto-restart (complexity: medium)
  - Add context % monitoring to taskmaster-wait.sh idle loop
  - At 70%: auto-send /compact
  - At 85%: kill and relaunch Claude process with fresh context + re-brief
  - Use existing context_pct status files as input

- [ ] Step 2: Hook output suppression (complexity: low)
  - on-prompt-submit.sh produces visible errors ("Unknown command: status")
  - Redirect hook stderr/stdout to /dev/null or runtime log
  - Ensure hook exit codes still propagate correctly

- [ ] Step 3: UserPromptSubmit hook hardening + integration test (complexity: medium)
  - 3rd time role detection broke workers (tasks #134, #136, #156)
  - Add tests/test-hook-worker-accept.sh
  - Add fail-open guard: if role detection fails, default to ALLOW for workers

### Wave 2 — Communication reliability
- [ ] Step 4: Unread message filtering (complexity: low)
  - Add --unread flag to doey msg read (Go code)
  - Track last-read message ID per pane in SQLite
  - Only return messages with id > last_read_id

- [ ] Step 5: Auto-nudge user-pane exclusion (complexity: low)
  - Add DOEY_USER_PANES env var (default: "0.1") to session.env
  - Auto-nudge checks exclusion list before send-keys
  - Extensible for future user-facing panes

- [ ] Step 6: Pane ID consistency (complexity: medium)
  - Audit formats: W.P vs session_W_P vs session:W.P
  - Pick canonical: W.P (tmux native)
  - Update status files + doey status get to resolve both

### Wave 3 — Coordination
- [ ] Step 7: Taskmaster phase auto-advance (complexity: medium)
  - When all workers FINISHED for a phase, Taskmaster auto-advances
  - Add phase tracking to taskmaster-wait.sh triggers
  - Consolidate, review, dispatch next phase without Boss nudge

## Risks
- Auto-restart losing context: must re-brief from .task files
- Pane ID migration: support both formats during transition
- Fail-open guard: only for UserPromptSubmit, pre-tool-use stays fail-closed

## Scope
- Files: taskmaster-wait.sh, on-prompt-submit.sh, Go code, session.env, status files
- Teams: 2 teams — Wave 1+2 parallel, Wave 3 after
- Workers: 4-6 total

## Acceptance Criteria
- Taskmaster auto-compacts at 70%, auto-restarts at 85%
- Workers never blocked by UserPromptSubmit (test proves it)
- Hook errors not visible to user
- doey msg read --unread returns only new messages
- Auto-nudge never sends keys to user-facing panes
- doey status get W.P works regardless of internal ID format
- Taskmaster advances phases within 30s of workers finishing
