# Task-Obsessed TUI — Live Activity Feed & Rich Task Lifecycle

## Goal
Make the Go TUI the live nerve center for all task activity. Surface subtask changes, status transitions, worker assignments, decisions, errors, and progress in real-time. The TUI should feel alive — every action in the system should be visible.

## Current State (What Exists)
- SQLite store already has: tasks, subtasks, task_log, events tables
- store.LogEvent() / store.ListEvents() exist but are UNUSED by the TUI model — events are written but never displayed
- store.AddTaskLog() / store.ListTaskLog() exist — task logs shown in REPORTS section in detail pane
- Subtasks render in detail pane with progress bar + status dots
- Task cards show title, status icon, subtask count — but no activity indicators
- runtime.StoreReader loads tasks+subtasks from SQLite on each tick
- MISSING: activity feed view, event timeline, consistent event emission from hooks

## Steps

### Phase 1: Event Emission (Shell Side) — complexity: medium
Enrich hooks to emit events to SQLite via doey-ctl on every meaningful lifecycle moment:
- 1a. Add "doey event log" CLI command to doey-ctl (or verify it exists)
- 1b. on-prompt-submit.sh: emit task_started event when BUSY + task assigned
- 1c. stop-status.sh: emit task_completed / task_failed events
- 1d. stop-results.sh: emit result_captured event with file count + tool stats
- 1e. Subtaskmaster dispatch: emit worker_dispatched event (pane, task, subtask)
- 1f. doey task create/update/subtask: emit corresponding events on every mutation

### Phase 2: Activity Feed Tab/Panel in TUI — complexity: high
Add a live activity feed that shows the stream of events across all tasks:
- 2a. Add ActivityModel in tui/internal/model/activity.go — scrollable list of recent events
- 2b. Wire store.ListEvents("", 100) into the model on each tick refresh
- 2c. Render each event as a compact line: [time] [icon] [source] message with color-coded types
- 2d. Add "Activity" tab to the tab bar (alongside Tasks, Plans, Teams, etc.)
- 2e. Click/enter on an activity entry navigates to the relevant task detail
- 2f. Filter by event type (status changes, dispatches, completions, errors)

### Phase 3: Enhanced Task Detail — Timeline Section — complexity: medium
Add a chronological timeline to the task detail right panel:
- 3a. Query store.ListEventsByTask(taskID) + store.ListTaskLog(taskID) and merge by timestamp
- 3b. Render as TIMELINE section in detail pane — newest at bottom, scrollable
- 3c. Each entry: colored type badge, timestamp, source pane, summary text
- 3d. Collapse/expand long entries (reports, decisions)

### Phase 4: Task Card Enrichment — complexity: low
Make task cards show more at-a-glance info:
- 4a. Show active worker count on card (e.g., "2 workers")
- 4b. Show latest activity timestamp ("3m ago")
- 4c. Subtask progress bar on card itself (compact)
- 4d. Animated/pulsing indicator for tasks with recent activity (within 30s)

### Phase 5: Subtask Deep Visibility — complexity: medium
- 5a. Subtasks clickable — show assigned worker, status history, output preview
- 5b. Parent-child task linking visible in task list (indented children or tree view toggle)
- 5c. Aggregate progress: parent task shows rolled-up completion % from children

## Risks
- Event volume: High-frequency events could bloat SQLite → cap at 1000 events, auto-prune old ones
- Tick performance: Loading events on every tick → cache with dirty flag, only re-query when mtime changes
- Shell hook overhead: Extra doey-ctl calls → fire-and-forget background calls
- Stale events: Events from crashed workers → event TTL, visual staleness indicator

## Scope
- Files: tui/internal/model/activity.go (new), tui/internal/model/tasks.go, tui/internal/model/root.go (add tab), tui/internal/taskcard/taskcard.go, tui/internal/store/events.go, tui/internal/runtime/store_reader.go, shell/doey-ctl event command, hooks (stop-status.sh, stop-results.sh, on-prompt-submit.sh)
- Teams: 2 teams — one for Go TUI work, one for shell/hook event emission
- Workers: 3-4 total (2 Go, 1-2 shell)

## Acceptance Criteria
- Activity feed tab shows real-time stream of task events
- Task detail pane has a TIMELINE section showing chronological event history
- Task cards show worker count, latest activity age, and subtask progress
- Events emitted by hooks on: task create, status change, subtask mutation, worker dispatch, completion, failure
- go build passes in tui/
- tests/test-bash-compat.sh passes for any shell changes
- Fresh install works — no manual setup required
