# Plan Mode Implementation

## Intent
Add a first-class planning system to Doey, inspired by Cursor's plan mode. Plans are rich markdown documents linked to tasks.

## Tasks
- [x] Research TUI architecture
- [x] Design plan storage format
- [ ] Implement shell CRUD helpers
- [ ] Create PlansModel for TUI
- [ ] Wire Plans tab into root
- [ ] Add glamour markdown rendering

## Architecture
Plans stored as markdown files with YAML frontmatter in `.doey/plans/`.
TUI renders plans in a 2-pane layout (list left, content right).
Shell helpers handle CRUD; Go TUI reads files directly.

## Constraints
- Bash 3.2 compatible for shell helpers
- No external dependencies beyond existing Charm ecosystem
- Plans must survive session restart (persisted to disk)

## Success Criteria
- Plans tab visible in TUI with working navigation
- Plan content renders with markdown formatting
- Plans link to tasks bidirectionally
