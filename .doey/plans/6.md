# Unified Data Layer

## Goal
Replace all scattered data stores (tasks, plans, teams, agents, logs, config, IPC) with a single SQLite database at .doey/doey.db. One Go library (internal/store/) manages all reads/writes. doey-ctl becomes the universal CLI. Shell helpers call doey-ctl instead of file manipulation.

## Current State (what we're replacing)
- .doey/tasks/*.task — custom key=value, fragile parsing
- .doey/plans/*.md — YAML frontmatter + markdown
- /tmp/doey/<project>/status/*.status — custom text
- /tmp/doey/<project>/messages/*.msg — custom text
- agents/*.md — YAML frontmatter (read-only definitions)
- .doey/tasks/*.result.json — JSON
- shell/doey-roles.sh, .doey/config.sh — bash variables

## Schema Design

### tasks
- id INTEGER PRIMARY KEY
- title TEXT NOT NULL
- status TEXT NOT NULL DEFAULT 'active'
- type TEXT (feature/bugfix/refactor/research/infrastructure/docs)
- description TEXT
- created_by TEXT
- assigned_to TEXT
- team TEXT
- plan_id INTEGER REFERENCES plans(id)
- tags TEXT (comma-separated)
- acceptance_criteria TEXT
- current_phase INTEGER DEFAULT 0
- total_phases INTEGER DEFAULT 0
- created_at INTEGER (unix timestamp)
- updated_at INTEGER

### subtasks
- id INTEGER PRIMARY KEY
- task_id INTEGER REFERENCES tasks(id)
- seq INTEGER (order within task)
- title TEXT NOT NULL
- status TEXT DEFAULT 'pending'

### task_log
- id INTEGER PRIMARY KEY
- task_id INTEGER REFERENCES tasks(id)
- type TEXT (decision/note/report/conversation/qa_thread/research_cycle)
- author TEXT
- title TEXT
- body TEXT
- created_at INTEGER

### plans
- id INTEGER PRIMARY KEY
- title TEXT NOT NULL
- status TEXT DEFAULT 'draft'
- body TEXT (full markdown content)
- created_at INTEGER
- updated_at INTEGER

### teams
- window_id TEXT PRIMARY KEY (e.g. "W1")
- name TEXT
- type TEXT (standard/freelancer/rd)
- worktree_path TEXT
- pane_count INTEGER
- created_at INTEGER

### pane_status
- pane_id TEXT PRIMARY KEY (e.g. "1_0")
- window_id TEXT REFERENCES teams(window_id)
- role TEXT (subtaskmaster/worker/freelancer)
- status TEXT (READY/BUSY/FINISHED/ERROR/RESERVED)
- task_id INTEGER
- task_title TEXT
- agent TEXT
- updated_at INTEGER

### messages
- id INTEGER PRIMARY KEY
- from_pane TEXT
- to_pane TEXT
- subject TEXT
- body TEXT
- task_id INTEGER
- read INTEGER DEFAULT 0
- created_at INTEGER

### events (trace log)
- id INTEGER PRIMARY KEY
- type TEXT (msg_sent/msg_read/status_change/task_update/phase_change/error)
- source TEXT
- target TEXT
- task_id INTEGER
- data TEXT (JSON blob)
- created_at INTEGER

### agents (read-only registry, populated on startup from agents/*.md)
- name TEXT PRIMARY KEY
- display_name TEXT
- model TEXT
- description TEXT
- file_path TEXT

### config
- key TEXT PRIMARY KEY
- value TEXT
- source TEXT (default/global/project)

## Steps

- [ ] Step 1: Create internal/store/ package — schema, migrations, CRUD for all tables (high)
- [ ] Step 2: Extend doey-ctl with store subcommands — task/plan/team/status/msg/event/config CRUD via SQLite (high)
- [ ] Step 3: Migration tool — doey-ctl migrate: reads existing .task/.md/status files, populates SQLite (medium)
- [ ] Step 4: Replace shell helpers — doey-task-helpers.sh and doey-ipc-helpers.sh call doey-ctl exclusively (medium)
- [ ] Step 5: Replace hook file operations — status reads/writes, msg reads/writes go through doey-ctl (medium)
- [ ] Step 6: TUI reads from SQLite — tui/internal/runtime/reader.go queries doey.db instead of parsing files (medium)
- [ ] Step 7: doey-router uses store directly — Go import, no file I/O for routine operations (medium)
- [ ] Step 8: Agent registry — on-session-start populates agents table from agents/*.md (low)
- [ ] Step 9: Verification — full session lifecycle, go test, bash compat, TUI renders correctly (medium)

## Risks
- SQLite locking under concurrent writes from multiple panes. Mitigation: WAL mode, short transactions, doey-ctl serializes writes
- Big bang migration breaks running sessions. Mitigation: run migration on fresh session start only
- Shell helpers depend on file existence checks. Mitigation: doey-ctl returns proper exit codes
- TUI import cycle (store vs runtime packages). Mitigation: clean interface boundaries

## Scope
- Files: new tui/internal/store/, extend tui/cmd/doey-ctl/, modify shell/*.sh, modify .claude/hooks/*.sh, modify tui/internal/runtime/
- Teams: 2 teams recommended (Go work + shell integration in parallel)
- Workers: 4-6 total

## Acceptance Criteria
- Single .doey/doey.db contains all state
- doey-ctl is the sole interface for all data operations
- No direct file parsing in shell helpers or hooks
- TUI renders from SQLite
- doey-router reads/writes through store package
- Migration from existing files works on first run
- WAL mode, no lock contention under normal load
- go build passes, bash compat passes, full session works
