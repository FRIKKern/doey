# Instant ASCII Loading Screen on Startup

## Goal
The ASCII loading screen (doey-loading) must appear immediately when the user runs doey. Currently it shows up after all the heavy setup (wizard, session creation, theme, grid, team spawn, banner) — or not at all. The user wants it to be the very first visual.

## Current Flow (Problem)
In launch_session_dynamic() (shell/doey.sh ~line 3623):
1. Banner prints (doey_banner + doey_info lines)
2. ensure_project_trusted + install_doey_hooks
3. Steps 1-5: session creation, theme, grid, Subtaskmaster, workers
4. Full "Doey is ready" gum banner prints
5. Background subshell spawns remaining teams
6. doey-loading runs here — too late, everything is already done
7. attach_or_switch — enters tmux

The doey-loading binary exists and works, but it is positioned after ALL setup completes, making it pointless — the user has already watched 10+ seconds of step output.

Same pattern in launch_session() (line 2084) — loading screen after full setup.

## Steps
- [ ] Step 1: Move doey-loading invocation to right after _init_doey_session (so --session flag works), BEFORE steps 2-5. Run remaining setup behind the loading screen. (complexity: medium)
- [ ] Step 2: Show ASCII art splash immediately as first output in launch_session_dynamic, before session creation. Transition to doey-loading once session exists. (complexity: medium)
- [ ] Step 3: Suppress step-by-step output when loading screen is active — no double output. Redirect to log or set a flag. (complexity: low)
- [ ] Step 4: Apply same fix to launch_session() (non-dynamic path). (complexity: low)
- [ ] Step 5: Handle reattach case — _attach_session currently shows nothing. (complexity: low)

## Risks
- doey-loading needs --session and --runtime flags, which require session to exist first. Mitigation: create session early, show loading screen, do remaining setup behind it.
- Loading screen polls for worker readiness — if started too early it may timeout. Mitigation: adjust timeout.
- Step output interleaving with loading screen TUI. Mitigation: suppress stdout during loading.

## Scope
- Files: shell/doey.sh (both launch_session and launch_session_dynamic), possibly tui/cmd/doey-loading/main.go
- Teams: 1 worker
- Workers: 1-2

## Acceptance Criteria
- Running doey shows ASCII art within 1 second
- No step-by-step output visible to user (happens behind loading screen)
- Loading screen transitions to tmux attach when ready
- Works for both dynamic and static grid paths
