# Deep Interview Brief: Improve the Masterplan Plan Pane

**Status:** Phase 5 — Synthesis (final, awaiting user approval)
**Masterplan ID:** masterplan-20260426-203854

## Intent
Make the Masterplan Plan pane the best live planning view in the industry — surpassing Claude Code's native planning and Cursor's planning view — by combining maximum Doey-native information density (live plan content, consensus state, research findings, task wiring) with elegant typographic craft (golden-ratio line length, horizontal centering, responsive layout). It must serve both the human observer (glanceable) and agent coordinators (machine-readable hand-off signals).

## Scope
- Plan pane only — same tmux pane slot in the Masterplan window. No changes to Planner or Researcher panes.
- New Go TUI binary `doey-plan-view` — sub-package of `tui/`, full layout control, ANSI styling, golden-ratio typography.
- v1 ships three data pillars: live plan markdown, consensus state header, research findings index.
- v1 prerequisites (non-negotiable, alongside the pillars): `--demo` mode (so user can sign off cheaply on fixture runs); install.sh wiring (so fresh-install survives — binary lands at `~/.local/bin/doey-plan-view`).
- v2 follow-up: task/phase wiring footer, plan diff highlights, mouse + key bindings, expanded demo fixtures.

## Non-Goals
- Not a redesign of the Planner or Researcher panes.
- Not a redesign of the Boss/Taskmaster panes or the Doey dashboard.
- Not a bash/info-panel-style live tail — the elegance bar requires Go TUI.
- Not a kitchen-sink dashboard — every element earns its place.
- Not the action_request hook fix (separate concern).

## Constraints
- **Fresh-install invariant (HARD):** must work after `curl|bash` with no manual setup. If `go` is not present, install fails loud or ships a prebuilt fallback — silent breakage is forbidden.
- **Refresh model:** hybrid — `fsnotify` on plan/consensus/research files for change-driven updates, plus a 1s heartbeat tick for clock/age fields. Idle CPU near zero.
- **Strong defaults (non-blockers, but tradeoffs favor):** Bash 3.2 compat for shell glue; pure Go + lipgloss/bubbletea-class deps only (no CGO, no exotic runtime deps); no edits to `~/.claude/settings.json` (self-install via on-session-start hook or install.sh).
- **Terminal coverage (verify pre-flight):** Linux + macOS terminals; truecolor + 256-color fallback; 80 / 120 / 200-col pane widths.
- **File-contract stability:** the pane reads runtime state files (plan markdown, consensus JSON, research dir, task DB). The contract becomes part of the design — changes to it are breaking changes.

## Success Criteria
- [ ] `doey-plan-view` binary builds during `install.sh` and lands at `~/.local/bin/doey-plan-view` on a fresh machine; `curl|bash` install path passes.
- [ ] `masterplan-spawn.sh` wires `doey-plan-view` into the Plan pane of the Masterplan window — replacing whatever runs there now.
- [ ] Live plan markdown renders with golden-ratio line length (~65 chars / phi-based measure), horizontal centering, and responsive layout at 80/120/200-col pane widths.
- [ ] Consensus state header shows: current round number, agreed parties, blocking parties, last-vote summary, time-since-last-update.
- [ ] Research findings index shows: file list with one-line abstracts and resolvable file paths.
- [ ] Hybrid refresh: `fsnotify` triggers re-render on plan/consensus/research file changes; 1s heartbeat tick updates clock/age fields; idle CPU < 1% on a quiescent run.
- [ ] `doey-plan-view --demo` runs against fixture data without a live masterplan session (for cheap aesthetic review and regression).
- [ ] Automated screenshot regression harness produces golden diffs at 80/120/200 cols on Linux truecolor + 256-color and exits 0 on match.
- [ ] Side-by-side smoke test against Claude Code native planning and Cursor planning view: user sign-off via `--demo` mode confirms the elegance bar is met.
- [ ] Pane remains functional under tmux passthrough — no broken right-click menu or scroll/copy-mode regression in the surrounding window.

## Recommended Approach
Build `doey-plan-view` as a Bubble Tea (lipgloss-styled) Go TUI. Use a layered renderer: a typographic baseline (centered viewport, golden-ratio measure, responsive breakpoints at 80/120/200 cols) is the foundation; a header zone hosts the consensus state; a body zone renders the plan markdown; a footer zone hosts the research index. Data layer is read-only file watchers (fsnotify) over a documented file contract (plan.md, consensus.json, research/*.md), with a 1s tick for age/clock fields. `--demo` mode injects fixture files instead of live runtime state, enabling cheap regression and aesthetic review. Install.sh adds a `go build` step (fail-loud if Go missing); masterplan-spawn.sh launches the binary in the Plan pane. Architect/Critic verify via consensus rounds; user signs off via demo runs.

## Risks & Mitigation
| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| Typography looks great in iTerm2 but breaks in Linux / non-truecolor terms | High | Pre-flight matrix on Linux + macOS, truecolor + 256-color, before consensus close. lipgloss adaptive color profile. |
| Golden-ratio centering wastes screen on narrow panes | Medium | Responsive breakpoints: 80-col uses full width with measure cap; 120/200-col centers with margin. |
| File-contract drift breaks the pane silently | High | Define `docs/plan-pane-contract.md` as source of truth. Add a contract validator to the install path. |
| Mouse/key bindings conflict with tmux passthrough | Medium | v2 only. v1 ships read-only — no input handling. Defer the conflict surface entirely. |
| Build regression on fresh install (no Go) | High | install.sh detects `go` and fails loud with a clear remediation message; optionally ship a prebuilt fallback for common arch/OS. |
| `fsnotify` flakiness on macOS / network mounts | Medium | Fall back to 1s polling on watcher error; surface a `WATCH: degraded` flag in the pane footer. |
| User taste call rejects on first review and rework cycle is expensive | Medium | `--demo` mode lets the Planner iterate on fixture data without a real masterplan run. Cheap rework. |
| Scope creep into Planner/Researcher panes | Medium | Non-Goals section is explicit. Critic enforces during consensus rounds. |
