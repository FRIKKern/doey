# Masterplan: doey.sh Decomposition

## Intent
Decompose the 6,209-line doey.sh monolith into ~12 focused shell modules, reducing doey.sh to a ~400-line thin dispatcher. Shell-only decomposition — Go migration deferred as separate decision. Every phase ships independently, fresh install works at every step.

## Phases
- Phase 0: Foundation — test harness, perf baseline, helpers + UI extraction (~430L)
- Phase 1: Remote + Purge extraction (~830L)
- Phase 2: Update/Install + Doctor extraction (~920L)
- Phase 3: Task CLI + Test Runner + dead code (~455L)
- Phase 4: Grid + Menu extraction (~370L)
- Phase 5: Global State Refactoring (internal, no new files)
- Phase 6: Team Management extraction (~720L)
- Phase 7: Session Launch Core (~800L, hardest)

## Dependencies
Phase 0 → Phases 1-4 (parallel) → Phase 5 → Phase 6 → Phase 7
Critical path: 0→5→6→7. Phase 7 skippable (80% value without it).
